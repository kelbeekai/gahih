# 가힣 포트폴리오 소개

## 1. 프로젝트 한 줄 소개

가힣는 해외 거주 한인과 한국에 관심 있는 사용자를 위한 국가별 커뮤니티 서비스입니다.

단순 게시판 CRUD를 넘어, 회원 상태, 신고, 관리자 조치, 운영 정책, 개발/운영 환경 분리, 배포 전 검증까지 직접 설계하고 구현한 Spring Boot 기반 포트폴리오 프로젝트입니다.

## 2. 프로젝트 목적

이 프로젝트는 “작지만 실제 운영 가능한 커뮤니티 서비스”를 목표로 개발했습니다.

단순히 게시글을 작성하고 조회하는 기능보다, 실제 서비스를 운영할 때 발생하는 문제를 직접 다루는 데 초점을 두었습니다.

주요 설계 고민은 다음과 같습니다.

* 회원이 탈퇴하거나 정지되었을 때 기존 콘텐츠를 어떻게 처리할 것인가
* 비밀글은 누구에게 어디까지 보여줄 것인가
* 신고 대상과 신고자를 어떻게 기록하고 관리할 것인가
* 관리자 조치는 어떤 단위로 기록하고 추적할 것인가
* 게시판별로 댓글, 반응, mention 정책을 어떻게 다르게 적용할 것인가
* 개발 환경과 운영 환경을 어떻게 분리하고 검증할 것인가

## 5. 중점 구현 포인트

## 5.1 회원 상태 기반 운영 정책

회원 상태를 `ACTIVE`, `SUSPENDED`, `WITHDRAWN`, `DELETED`로 구분하고, 상태에 따라 서비스 접근 범위를 다르게 처리했습니다.

정지 회원은 일반 활동을 제한하고, 탈퇴 회원은 유예 기간 이후 최종 종료될 수 있도록 설계했습니다. 탈퇴 또는 삭제된 회원의 콘텐츠와 닉네임 노출도 별도 정책으로 분리했습니다.

## 5.2 비밀글과 이용문의 정책

이용문의 게시판에서는 비밀글을 지원하고, 비밀글은 작성자와 관리자 중심으로 열람 권한을 제한했습니다.

비밀글을 볼 수 없는 사용자는 본문, 댓글, mention 후보를 볼 수 없도록 처리했습니다. 이를 통해 단순 화면 숨김이 아니라 서버에서 내려주는 데이터 자체를 제한하는 방향으로 구현했습니다.

## 5.3 게시판별 댓글 / 반응 / mention 정책

게시판 성격에 따라 댓글과 반응 정책을 다르게 적용했습니다.

예를 들어 구인구직 게시판은 사업주, 고용주, 구직자 간 분쟁과 비방을 줄이기 위해 댓글과 반응을 제한했습니다. 댓글이 허용되지 않는 게시판에서는 mention도 함께 비활성화되도록 정책을 통일했습니다.

mention 기능은 다음 정책을 반영했습니다.

* 댓글 허용 게시판에서만 mention 가능
* 구인구직 게시판에서는 mention 불가
* 비밀글에서는 열람 권한에 따라 mention 제한
* 일반 사용자의 관리자 mention은 이용문의 중심으로 제한
* 관리자는 댓글 허용 게시판에서 다른 관리자를 mention 가능
* 자기 자신 mention은 제한

## 5.4 신고와 관리자 조치

게시글, 댓글, 첨부파일, 작성자 신고를 분리해 처리했습니다.

신고는 대상별로 기록되며, 중복 신고를 제한했습니다. 관리자는 신고 내용을 확인한 뒤 블라인드, 복구, 삭제 등의 조치를 수행할 수 있고, 조치 사유와 당시 상태를 관리자 로그에 기록하도록 구현했습니다.

이를 통해 단순 관리자 기능이 아니라, 사후 추적이 가능한 운영 흐름을 구성했습니다.

## 5.5 개발 환경과 운영 환경 분리

개발 환경은 H2와 테스트 데이터를 기반으로 빠르게 화면과 기능을 확인할 수 있도록 구성했습니다.

운영 환경은 MySQL과 `prod` 프로필을 사용하며, Hibernate `ddl-auto=validate`로 운영 스키마와 엔티티 매핑이 일치하는지 검증하도록 구성했습니다.

비밀값은 환경변수로 분리했습니다.

* DB URL
* DB 계정
* 메일 계정
* 관리자 초기 계정
* 파일 업로드 경로

## 7. 문제 해결 사례

### 7.1 잘못된 path variable 요청 처리

`/c/DE/posts/abc`처럼 숫자 ID가 필요한 경로에 문자열이 들어오는 경우 기존에는 500 오류로 처리될 수 있었습니다.

이를 서버 장애가 아니라 잘못된 요청으로 분류하기 위해 `MethodArgumentTypeMismatchException`을 전역 예외 처리기에서 400 오류로 처리하도록 개선했습니다.

### 7.2 오류 페이지 헤더 컨텍스트 개선

오류 페이지에서는 로그인 상태와 국가 선택 드롭다운이 정상적으로 반영되지 않는 문제가 있었습니다.

전역 예외 처리기에서 오류 페이지용 공통 모델을 구성하도록 변경하고, 오류 템플릿에서 로그인 회원 정보를 헤더에 전달하도록 수정했습니다.

### 7.3 mention 정책 보강

관리자 mention, 비밀글 mention, 댓글 불가 게시판 mention 정책을 테스트 중 발견한 문제에 맞춰 보강했습니다.

화면에서 버튼을 숨기는 것뿐 아니라, 서버에서 mention 후보 생성과 mention 저장을 함께 제한해 우회 입력 가능성을 줄였습니다.
