# 가힣 프로젝트 개요

## 1. 프로젝트 소개

가힣는 해외 거주 한인과 한국에 관심 있는 사용자를 위한 국가별 커뮤니티 서비스이다.

초기 버전은 독일과 일본 커뮤니티를 중심으로 구성하며, 회원 인증, 게시글, 댓글, 첨부파일, 신고, 관리자 운영 기능을 포함한 실제 운영 가능한 게시판형 서비스를 목표로 한다.

이 프로젝트는 단순 CRUD 구현이 아니라, 운영 정책, 회원 상태, 신고 처리, 관리자 조치, 배포 환경 분리까지 포함한 실서비스형 포트폴리오 프로젝트이다.

## 2. 1차 버전 목표

1차 버전의 목표는 다음과 같다.

* 국가별 커뮤니티 구조 구현
* 회원가입 / 로그인 / 로그아웃 구현
* 이메일 인증 기반 회원가입 구현
* 아이디 찾기 / 비밀번호 찾기 구현
* 내 정보 수정 / 비밀번호 변경 / 회원 탈퇴 구현
* 게시글 CRUD 구현
* 댓글 CRUD 구현
* 게시글 / 댓글 반응 구현
* 첨부파일 업로드 / 다운로드 구현
* 비밀글 / 고정글 / 구인구직 댓글 제한 정책 구현
* 신고 기능 구현
* 관리자 페이지 구현
* 관리자 로그 기록 구현
* 개발 환경과 운영 환경 분리
* 운영 DB 스키마 검증
* 배포 전 수동 회귀 테스트 수행

## 3. 프로젝트 핵심 가치

이 프로젝트의 핵심 가치는 다음과 같다.

* 실제 서비스 운영을 고려한 게시판 구조 설계
* 인증과 인가 흐름을 직접 구현
* 회원 상태에 따른 서비스 접근 정책 구현
* 국가별 커뮤니티 라우팅 구조 설계
* 관리자 조치와 로그 기록을 통한 운영 가능성 확보
* 신고, 블라인드, 삭제, 탈퇴, 정지 등 운영 정책 설계
* 개발 환경과 운영 환경을 명확히 분리
* 배포 전 검증 절차를 문서화

## 6. 주요 도메인

### 회원

회원 도메인은 다음 기능을 포함한다.

* 회원가입
* 로그인 / 로그아웃
* 이메일 인증
* 아이디 찾기
* 비밀번호 재설정
* 내 정보 수정
* 비밀번호 변경
* 회원 탈퇴
* 닉네임 변경 제한
* 회원 상태 관리

회원 상태는 다음 흐름을 고려한다.

* ACTIVE
* SUSPENDED
* WITHDRAWN
* DELETED

### 게시글

게시글 도메인은 다음 기능을 포함한다.

* 작성 / 조회 / 수정 / 삭제
* 카테고리별 목록
* 국가별 목록
* 검색 / 정렬 / 페이지네이션
* 고정글
* 비밀글
* 첨부파일
* 조회수
* 신고
* 관리자 조치

### 댓글

댓글 도메인은 다음 기능을 포함한다.

* 작성 / 수정 / 삭제
* 정렬 / 페이지네이션
* 좋아요 / 싫어요
* mention
* 신고
* 관리자 조치

댓글과 mention은 게시판 정책에 따라 제한된다.

* 댓글 허용 게시판에서만 mention 가능
* 구인구직 게시판은 댓글과 mention 불가
* 비밀글에서는 열람 권한에 따라 mention 제한
* 일반 게시판에서는 일반 사용자의 관리자 mention 제한
* 관리자는 댓글 허용 게시판에서 다른 관리자를 mention 가능

### 신고

신고 도메인은 다음 기능을 포함한다.

* 게시글 신고
* 댓글 신고
* 첨부파일 신고
* 작성자 신고
* 중복 신고 제한
* 신고 처리
* 신고 대상 상태 관리
* 신고자 활동 기록

### 관리자

관리자 기능은 다음 흐름을 포함한다.

* 게시글 관리
* 댓글 관리
* 회원 관리
* 신고 관리
* 관리자 로그 조회
* 관리자 조치 사유 입력
* 관리자 로그 snapshot 기록
* 국가별 관리자 화면
* 전역 관리자 모니터링 화면

## 7. 운영 정책

프로젝트에서 직접 설계한 주요 운영 정책은 다음과 같다.

### 비밀글 정책

비밀글은 작성자와 관리자 중심으로 열람 권한을 제한한다.
비밀글을 볼 수 없는 사용자는 본문, 댓글, mention 후보를 볼 수 없다.

### 구인구직 게시판 정책

구인구직 게시판은 사업주, 고용주, 구직자 간 분쟁과 비방을 줄이기 위해 댓글과 반응을 제한한다.
신고 기능은 유지하여 문제가 있는 게시글은 관리자에게 전달할 수 있다.

### 회원 상태 정책

회원 상태에 따라 서비스 접근 범위가 달라진다.

* 정지 회원은 일반 활동이 제한된다.
* 탈퇴 회원은 유예 기간 이후 최종 종료될 수 있다.
* 삭제 또는 탈퇴된 회원의 콘텐츠 노출 방식은 별도 정책에 따라 처리된다.

### 신고 및 관리자 조치 정책

신고는 대상별로 기록되며, 관리자는 신고 내용을 확인한 뒤 조치를 수행할 수 있다.
관리자 조치는 사유와 함께 기록되며, 관리자 로그에서 추적할 수 있다.
