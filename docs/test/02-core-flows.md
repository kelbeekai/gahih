# 가힣 핵심 플로우 정리

이 문서는 최종 회귀 테스트에서 어떤 사용자 여정을 우선 검증할지 정리한 문서이다.

## 1. 비로그인 방문자 플로우

### 1.1 국가 선택과 공개 탐색

1. `/` 접속.
2. 국가 선택 게이트에서 독일/일본 커뮤니티가 보이는지 확인.
3. 독일 커뮤니티 `/c/DE` 진입.
4. 일본 커뮤니티 `/c/JP` 진입.
5. 헤더 국가 드롭다운에서 국가 전환.
6. 각 국가의 게시판 탭/카테고리 목록 확인.
7. 게시글 목록 접근.
8. 공개 게시글 상세 접근.
9. 첨부 이미지 미리보기와 다운로드 링크 표시 확인.
10. 푸터 정책 페이지 접근.

### 1.2 비로그인 제한 확인

1. 글쓰기 버튼 클릭 시 로그인 화면으로 이동.
2. 댓글 작성 시도 시 로그인 화면으로 이동.
3. 좋아요/싫어요 시도 시 로그인 화면으로 이동.
4. 신고 시도 시 로그인 화면으로 이동.
5. 마이페이지 접근 시 로그인 화면으로 이동.
6. 관리자 화면 접근 시 로그인 화면으로 이동.

## 2. 일반 회원 플로우

### 2.1 회원가입

1. 회원가입 화면 진입.
2. 필수 동의 모달 3개 확인.
3. 동의하지 않은 상태에서 가입 제출 차단 확인.
4. 이용약관/개인정보/운영정책 모달 각각 열기.
5. 모달 내부 체크 시 외부 체크박스 반영 확인.
6. 이메일 인증번호 발송.
7. 잘못된 인증번호 입력 시 오류 확인.
8. 올바른 인증번호 인증.
9. 아이디/비밀번호/닉네임/이메일 입력.
10. 가입 완료.

### 2.2 로그인/로그아웃

1. 가입한 회원으로 로그인.
2. 로그인 후 헤더가 회원 상태로 바뀌는지 확인.
3. 마이페이지 진입.
4. 로그아웃.
5. 로그아웃 후 헤더가 비로그인 상태로 돌아오는지 확인.

### 2.3 계정 복구

1. 아이디 찾기 화면 진입.
2. 이메일 인증번호 발송/검증.
3. 아이디 표시 또는 마스킹 표시 확인.
4. 비밀번호 찾기 화면 진입.
5. 이메일 인증번호 발송/검증.
6. 비밀번호 재설정 화면으로 이동.
7. 새 비밀번호 설정.
8. 새 비밀번호로 로그인.

### 2.4 마이페이지/계정 관리

1. 마이페이지 진입.
2. 내 활동 통계 표시 확인.
3. 최근 반응/멘션 카드 표시 확인.
4. 내가 쓴 글 목록 이동.
5. 프로필 수정 화면 진입.
6. 닉네임 변경.
7. 같은 닉네임/중복 닉네임/정책 위반 닉네임 오류 확인.
8. 이메일 변경 인증 흐름 확인.
9. 비밀번호 변경.
10. 변경된 비밀번호로 재로그인.

### 2.5 게시글 작성/수정/삭제

1. 일반 카테고리에서 글쓰기.
2. 제목/본문 누락 검증 확인.
3. 첨부파일 없이 작성.
4. 작성 직후 상세 화면 진입.
5. 새로고침 시 조회수 이상 증가 여부 확인.
6. 수정 화면 진입.
7. 제목/본문 수정.
8. 수정됨 표시 확인.
9. 삭제.
10. 삭제 후 목록/상세 접근 결과 확인.

### 2.6 첨부파일

1. 이미지 파일 첨부 후 글 작성.
2. 상세에서 이미지 미리보기 확인.
3. 파일명/용량/다운로드 수 표시 확인.
4. 개별 다운로드.
5. ZIP 다운로드.
6. 같은 세션 반복 다운로드 시 카운트 과도 증가 여부 확인.
7. 수정 화면에서 기존 첨부 삭제.
8. 신규 첨부 추가.
9. 3개 초과, 단일 10MB 초과, 총 30MB 초과 검증 확인.

### 2.7 댓글/반응/멘션

1. 댓글 허용 카테고리 글 상세 진입.
2. 댓글 작성.
3. 댓글 수정.
4. 댓글 삭제.
5. 댓글 정렬 변경.
6. 댓글 페이지 이동.
7. 게시글 좋아요/싫어요 클릭.
8. 같은 버튼 재클릭 시 취소 확인.
9. 반대 반응 클릭 시 전환 확인.
10. 댓글 좋아요/싫어요도 동일하게 확인.
11. 작성자 멘션 버튼이 적절히 표시되는지 확인.

### 2.8 신고

1. 다른 회원이 작성한 게시글 신고.
2. 같은 게시글 재신고 차단 확인.
3. 다른 회원 댓글 신고.
4. 첨부파일 신고.
5. 작성자/회원 신고.
6. 본인 글/본인 댓글/본인 계정 신고 버튼 미노출 또는 차단 확인.
7. 관리자 작성 대상 신고 제한 확인.

## 3. 국가별 커뮤니티 플로우

### 3.1 국가별 데이터 분리

1. `/c/DE`에서 게시글 목록 확인.
2. `/c/JP`에서 게시글 목록 확인.
3. 독일 카테고리로 작성한 글이 일본 목록에 섞이지 않는지 확인.
4. 일본 카테고리로 작성한 글이 독일 목록에 섞이지 않는지 확인.
5. 상세 URL의 communityCode를 다른 국가 코드로 바꾸었을 때 접근이 차단되거나 404 처리되는지 확인.

### 3.2 국가별 관리자 플로우

1. `/c/DE/admin` 진입.
2. 독일 게시글/댓글/신고 관리 화면 확인.
3. `/c/JP/admin` 진입.
4. 일본 게시글/댓글/신고 관리 화면 확인.
5. 국가별 관리 목록에서 다른 국가 데이터가 섞이지 않는지 확인.

## 4. 관리자 플로우

### 4.1 관리자 로그인/접근

1. 일반 회원으로 `/admin` 접근 시 차단 확인.
2. 관리자로 로그인.
3. `/admin` 접근.
4. 전역 관리자 홈에서 국가별 관리자 홈으로 이동.

### 4.2 게시글 관리

1. 전역 게시글 목록 검색/필터/정렬/페이지 이동.
2. 국가별 게시글 목록 검색/필터/정렬/페이지 이동.
3. 게시글 고정.
4. 고정 해제.
5. 블라인드.
6. 블라인드 후 일반 회원 상세 접근 차단 확인.
7. 관리자 원문 확인 가능 여부 확인.
8. 복구.
9. 삭제.
10. 영구삭제.
11. 조치 후 목록 검색 조건 유지 확인.
12. 상세에서 조치 후 상세로 복귀 확인.

### 4.3 댓글 관리

1. 전역 댓글 목록 검색/정렬/페이지 이동.
2. 국가별 댓글 목록 검색/정렬/페이지 이동.
3. 댓글 블라인드.
4. 복구.
5. 삭제.
6. 영구삭제.
7. 조치 후 원래 목록 조건 유지.

### 4.4 회원 관리

1. 회원 목록 검색.
2. role/status 필터.
3. 회원 상세 진입.
4. 권한 변경.
5. 기간정지.
6. 정지 회원 로그인/접근 제한 확인.
7. 정지 해제 또는 활성화.
8. 영구정지.
9. 닉네임 강제 변경.
10. 최종 종료.
11. 자기 자신에 대한 조치 버튼 제한 확인.

### 4.5 신고 관리

1. 신고 목록 확인.
2. 신고 대상별 이동.
3. 신고자 활동 목록 확인.
4. 조치 필요 없음 처리.
5. 처리 후 상태 변경 확인.
6. 동일 대상 신규 신고 회차 흐름 확인.

### 4.6 관리자 로그

1. 게시글/댓글/회원/신고 조치 후 관리자 로그 확인.
2. 로그 검색.
3. 기간 필터.
4. 대상 타입 필터.
5. 정렬/페이지 이동.
6. 변경 전/후 스냅샷 모달 확인.
7. 조치 사유 표시 확인.

## 5. 회원 상태 플로우

### 5.1 정지 회원

1. 관리자가 일반 회원을 기간정지.
2. 정지 회원으로 로그인.
3. 정지 안내 화면 확인.
4. 일반 게시판 글쓰기/댓글/반응 차단 확인.
5. 이용문의 접근/작성 가능 여부 확인.
6. 마이페이지 접근 가능 여부 확인.
7. 로그아웃 가능 여부 확인.

### 5.2 탈퇴 유예 회원

1. 회원이 탈퇴 신청.
2. 탈퇴 유예 안내 화면 확인.
3. 일반 기능 접근 시 탈퇴 유예 화면으로 이동하는지 확인.
4. 탈퇴 복구.
5. 복구 후 일반 기능 사용 가능 확인.

## 6. 운영 문서/정책 플로우

1. 푸터의 이용약관 페이지 접근.
2. 개인정보 처리방침 페이지 접근.
3. 운영정책 페이지 접근.
4. 면책조항 페이지 접근.
5. 운영문의 페이지 접근.
6. 개인정보 처리방침 표 가독성 확인.
7. 회원가입 모달 요약과 푸터 원문이 서로 모순되지 않는지 확인.

## 7. 배포/실행 플로우

1. `prod-like` MySQL DB 준비.
2. `deploy/db/schema.sql` import.
3. `prod` 프로필 `ddl-auto=validate`로 실행.
4. 루트, 독일, 일본 커뮤니티 접근.
5. 관리자 계정 생성 확인.
6. `clean bootJar` 성공 확인.
7. 최종 Git 상태 clean 확인.
