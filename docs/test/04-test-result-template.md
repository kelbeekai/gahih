### ISSUE-001: 중복 이메일 인증 요청 시 사용자 안내가 애매함

- 우선순위: P1
- 영역: 회원가입 / 이메일 인증 / UX 정책
- 테스트 항목:
    - P0-05 중복 이메일 차단
    - P0-06 회원가입 이메일 인증
- 현재 동작:
    - 이미 가입된 이메일로 인증번호 발송을 요청하면 가입 여부 노출 방지를 위해 일반 성공 메시지가 표시된다.
    - 실제 인증 메일은 발송되지 않는다.
    - 따라서 해당 이메일로는 인증 완료가 불가능하고 최종 회원가입도 완료할 수 없다.
- 판단:
    - 중복 이메일로 최종 가입이 불가능하다면 보안상 치명적 버그는 아니다.
    - 다만 사용자는 메일이 오지 않는 이유를 알기 어려울 수 있다.
- 개선안:
    - 인증번호 발송 안내 문구에 중립 안내를 추가한다.
    - 예: “메일이 보이지 않는 경우 스팸함을 확인하거나 이메일 주소를 다시 확인해주세요. 이미 가입한 적이 있다면 아이디 찾기 또는 비밀번호 찾기를 이용해주세요.”
- 처리 상태:
    - P0 테스트 이후 개선 예정

### ISSUE-002: 회원가입 제출 실패 후 인증코드 입력칸이 비어 보여 혼동될 수 있음

- 우선순위: P1
- 영역: 회원가입 / 이메일 인증 / UX
- 현재 동작:
    - 이메일 인증 완료 후 아이디/닉네임 중복 등 다른 입력 오류로 제출이 실패하면 비밀번호, 비밀번호 확인, 인증코드 입력칸이 비어 보인다.
    - 이메일 인증 완료 문구는 유지된다.
- 판단:
    - 비밀번호가 다시 표시되지 않는 것은 자연스럽다.
    - 인증 완료 상태가 유지된다면 기능상 문제는 아니다.
    - 다만 사용자가 인증을 다시 해야 하는지 혼동할 수 있다.
- 개선안:
    - 인증 완료 안내 문구를 보강한다.
    - 예: “이메일 인증이 완료되었습니다. 입력 오류가 있더라도 인증은 유지됩니다.”
- 처리 상태:
    - P0 테스트 이후 개선 예정
