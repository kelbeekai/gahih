# 가힣 최종 회귀 테스트 결과 기록

## 1. 테스트 개요

- 테스트 유형: 최종 배포 전 수동 회귀 테스트
- 테스트 환경: prod-like MySQL validate 환경
- 실행 방식: `prod` 프로필 `bootRun`, `clean bootJar`, 생성된 jar의 `java -jar` 실행
- 테스트 브라우저: Chrome 기준
- 테스트 기준 문서: `docs/test/03-final-regression-checklist.md`

## 3. 배포 차단 여부 판단

현재 발견된 항목들은 대부분 정책/UX/오류 처리 개선 사항이다. 핵심 기능이 전면적으로 중단되거나 데이터 정합성을 즉시 깨뜨리는 치명적 장애는 발견되지 않았다.

다만 아래 항목은 배포 전 마감 품질을 위해 순차적으로 해결하는 것을 권장한다.

1. 관리자 태그 제한 정책 불일치
2. 잘못된 게시글 식별자 요청이 500으로 표시되는 문제
3. 오류 페이지 헤더가 로그인/국가/방문자 컨텍스트를 반영하지 않는 문제
4. 회원가입 중복 이메일/인증 안내 UX 개선
5. 마이페이지 언급 댓글 카드 가독성 개선
6. 내 정보 수정 안내 카드 추가

## 4. 발견 이슈

### ISSUE-001. 관리자 계정이 일반 게시판 댓글에서 태그되는 문제

- 우선순위: P1 / 기능 정책 버그
- 영역: 댓글 태그, 관리자 계정 보호, 마이페이지 언급 댓글
- 현재 동작:
  - 관리자 닉네임 클릭으로 댓글 입력창에 태그를 넣을 수 있다.
  - 댓글에 관리자 닉네임을 직접 입력해도 태그로 인식되고 제출된다.
- 기대 동작:
  - 관리자 계정은 이용문의 게시판에서만 태그 가능해야 한다.
  - 일반 게시판에서는 관리자 닉네임 버튼이 노출되지 않아야 한다.
  - 일반 게시판에서 관리자 닉네임을 직접 입력해도 태그 엔티티로 저장되거나 태그 스타일로 렌더링되면 안 된다.
- 판단:
  - 기능 전체를 막는 장애는 아니지만, 기존 운영 정책과 다르다.
  - 마이페이지 언급 댓글과 운영자 알림 성격에도 영향을 줄 수 있으므로 우선 해결한다.
- 처리 상태: 미해결

### ISSUE-002. 오류 페이지 헤더에 로그인/방문자/국가 컨텍스트가 반영되지 않음

- 우선순위: P1 / UX 및 공통 레이아웃 정합성
- 영역: 오류 페이지, 공통 헤더, 로그인 세션, 국가 커뮤니티 컨텍스트
- 현재 동작:
  - 로그인 상태에서도 오류 페이지 헤더에 로그인/회원가입 버튼이 표시된다.
  - TODAY/TOTAL 방문자 수가 표시되지 않는다.
  - 국가 선택/국가 커뮤니티 상태가 최신 헤더와 일치하지 않는다.
- 기대 동작:
  - 오류 페이지에서도 로그인 세션이 있으면 닉네임/회원 메뉴가 표시된다.
  - 최신 헤더 정책과 동일하게 방문자 수, 국가 선택, 국가 커뮤니티 컨텍스트가 반영된다.
- 판단:
  - 핵심 기능 장애는 아니지만, 실제 서비스 완성도를 낮춘다.
  - 템플릿만이 아니라 오류 컨트롤러/모델 구성도 함께 확인해야 한다.
- 처리 상태: 미해결

### ISSUE-003. `/c/DE/posts/abc` 요청 시 400이 아니라 500으로 표시됨

- 우선순위: P1 / 오류 처리 개선
- 영역: 게시글 상세 라우팅, 타입 변환 오류, 공통 예외 처리
- 재현 경로:
  - `/c/DE/posts/abc`
- 현재 동작:
  - `MethodArgumentTypeMismatchException`이 발생한다.
  - 화면상 500 오류로 표시되고 터미널에 긴 오류 로그가 출력된다.
- 기대 동작:
  - 잘못된 게시글 식별자 형식은 400 Bad Request 또는 안전한 공통 오류 페이지로 처리되어야 한다.
  - 불필요하게 서버 내부 오류처럼 보이면 안 된다.
- 판단:
  - 서버가 다운되는 위험은 아니지만, 잘못된 사용자 입력/URL 조작이 500으로 보이는 것은 개선이 필요하다.
- 처리 상태: 미해결

### ISSUE-004. 마이페이지 나를 언급한 댓글 카드 가독성 부족

- 우선순위: P2 / UI 개선
- 영역: 마이페이지, 최근 반응/멘션 카드
- 현재 동작:
  - 나를 언급한 댓글 카드의 컬럼들이 한곳에 다닥다닥 붙어 보인다.
  - 화면과 카드 폭에 비해 테이블 내용 구분이 약하다.
- 기대 동작:
  - 실선 테이블 느낌은 피하되, 왼쪽/가운데/오른쪽 컬럼의 시각적 구분이 명확해야 한다.
  - 게시글 제목, 댓글 미리보기, 날짜/시간이 적절한 폭과 간격을 가져야 한다.
- 판단:
  - 기능 문제는 아니며, 회귀 테스트 이후 UI polish 대상으로 처리한다.
- 처리 상태: 미해결

### ISSUE-005. 내 정보 수정 화면에서 이메일 인증 범위가 혼동될 수 있음

- 우선순위: P2 / UX 개선
- 영역: 마이페이지, 회원정보 수정
- 현재 동작:
  - 닉네임, 이메일, 인증코드, 비밀번호 입력칸이 함께 표시된다.
  - 닉네임만 변경하는 경우에도 이메일 인증이 필요한 것처럼 느껴질 수 있다.
- 기대 동작:
  - 이메일 인증코드는 이메일 변경 시에만 필요하다는 안내가 있어야 한다.
- 개선 문구 후보:
  - 회원님의 닉네임과 이메일을 변경하실 수 있습니다.
  - 닉네임 변경은 30일에 한 번만 가능합니다.
  - 인증코드는 이메일 변경 시에만 입력해주세요.
- 처리 상태: 미해결

### ISSUE-006. 회원가입 중복 이메일/인증 안내 UX 개선 필요

- 우선순위: P1 / 정책 및 UX 개선
- 영역: 회원가입, 이메일 인증, 중복 이메일 정책
- 현재 동작:
  - 이미 가입된 이메일로 인증번호 발송을 요청해도 가입 여부 노출 방지 정책에 따라 일반 성공 메시지가 표시된다.
  - 실제 인증 메일은 발송되지 않으므로 사용자는 왜 메일이 오지 않는지 혼동할 수 있다.
  - 아이디/닉네임 중복 등으로 제출 실패 시 비밀번호/인증코드 입력칸은 비지만, 이메일 인증 완료 상태는 유지된다.
- 기대 동작:
  - 이메일 가입 여부를 직접 노출하지 않으면서도 사용자가 다음 행동을 이해할 수 있어야 한다.
- 개선 문구 후보:
  - 메일이 보이지 않는 경우 스팸함을 확인하거나 이메일 주소를 다시 확인해주세요.
  - 이미 가입한 적이 있다면 아이디 찾기 또는 비밀번호 찾기를 이용해주세요.
  - 이메일 인증이 완료되었습니다. 입력 오류가 있더라도 인증은 유지됩니다.
- 처리 상태: 미해결
