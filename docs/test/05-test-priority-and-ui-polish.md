# 가힣 회귀 테스트 후 개선 우선순위 및 처리 계획

## 1. 처리 원칙

현재 단계에서는 신규 기능을 추가하지 않는다. 회귀 테스트에서 발견된 문제 중 아래 범주만 처리한다.

- 기존 정책과 다른 동작 복구
- 잘못된 오류 처리 수정
- 사용자가 오해하기 쉬운 안내 문구 보강
- 화면 가독성 개선
- 배포/포트폴리오 신뢰도를 떨어뜨리는 문서·UI 정합성 개선

## 2. 우선순위 분류

| 순위 | 이슈 | 분류 | 이유 |
| --- | --- | --- | --- |
| 1 | 관리자 태그 제한 정책 복구 | P1 기능 정책 버그 | 기존 정책과 다른 동작이며 댓글/멘션 데이터에 영향을 준다. |
| 2 | 잘못된 게시글 ID 문자열의 500 처리 개선 | P1 오류 처리 | 사용자의 잘못된 URL이 서버 내부 오류처럼 보이면 안 된다. |
| 3 | 오류 페이지 헤더 컨텍스트 최신화 | P1 공통 UX | 로그인/국가/방문자 컨텍스트가 최신 공통 헤더와 다르다. |
| 4 | 회원가입 이메일 인증 안내 개선 | P1 정책/UX | 가입 여부 노출 방지 정책은 유지하되 사용자 혼동을 줄여야 한다. |
| 5 | 내 정보 수정 안내 카드 추가 | P2 UX | 이메일 변경 시에만 인증코드가 필요하다는 안내가 필요하다. |
| 6 | 마이페이지 언급 댓글 카드 가독성 개선 | P2 UI | 기능은 정상이나 카드 내 컬럼 구분이 약하다. |

## 3. 첫 번째 작업: 관리자 태그 제한 정책 복구

### 목표

관리자 계정은 이용문의 게시판에서만 태그 가능해야 한다. 일반 게시판에서는 관리자 닉네임을 클릭하거나 직접 입력해도 태그로 인식되지 않아야 한다.

### 확인할 코드 영역

- `CommentMentionService`
  - 댓글 저장 시 실제 태그 엔티티 생성 여부를 결정하는 핵심 로직
  - 관리자 계정이 허용 대상에 들어가는지 확인 필요
- `PostDetailService`
  - 게시글 작성자 mention 버튼 노출 여부
- `CommentListService`
  - 댓글 작성자 mention 버튼 노출 여부
- `PostDetailController`
  - `mentionableNicknames` 모델 구성
- `post-detail-comment-form.html`
  - 프론트 검증용 mention targets 전달
- `comment.js`
  - mention 입력/검증/상태 표시

### 기대 수정 방향

- 프론트 버튼 노출만 막지 않는다.
- 서버 저장 로직에서도 관리자 계정을 일반 게시판 mention 대상에서 제외한다.
- 이용문의 게시판에서는 정책에 따라 관리자 태그를 허용한다.
- 일반 게시판에서 `@관리자닉네임`을 직접 입력해도 일반 텍스트로만 남고, mention 엔티티가 생성되지 않아야 한다.

### 수정 후 재테스트

- 일반 게시판에서 관리자 닉네임 버튼이 보이지 않는다.
- 일반 게시판 댓글에 관리자 닉네임 직접 입력 후 저장해도 태그 스타일이 적용되지 않는다.
- 관리자 마이페이지의 나를 언급한 댓글에 해당 댓글이 쌓이지 않는다.
- 이용문의 게시판에서는 관리자 태그가 가능하다.
- 일반 회원 간 태그는 기존처럼 정상 동작한다.

## 4. 두 번째 작업: 잘못된 게시글 ID 문자열 오류 처리

### 목표

`/c/DE/posts/abc` 같은 요청이 500이 아니라 400 또는 안전한 공통 오류 페이지로 처리되도록 한다.

### 기대 수정 방향

- `MethodArgumentTypeMismatchException`을 공통 예외 처리에서 잡는다.
- 잘못된 URL 파라미터/PathVariable 타입은 400 Bad Request로 처리한다.
- 운영 로그에는 간결한 warn 수준 로그만 남기고 긴 stack trace는 불필요하게 노출하지 않는다.

## 5. 세 번째 작업: 오류 페이지 헤더 최신화

### 목표

오류 페이지에서도 최신 공통 헤더와 동일하게 로그인 상태, 방문자 수, 국가 컨텍스트가 반영되도록 한다.

### 확인할 코드 영역

- 오류 컨트롤러 또는 공통 예외 처리기
- 에러 템플릿
- 헤더 fragment 호출 방식
- 로그인 회원 모델 attribute 구성
- 국가 커뮤니티 목록/현재 국가 모델 attribute 구성
- TODAY/TOTAL 방문자 수 모델 attribute 구성

## 6. 네 번째 작업: 회원가입 이메일 인증 안내 개선

### 목표

가입 여부 노출 방지 정책은 유지하면서, 메일이 오지 않는 사용자가 다음 행동을 이해할 수 있도록 한다.

### 문구 후보

```text
입력하신 이메일로 인증 안내 메일을 보냈습니다. 메일함을 확인해주세요.
메일이 보이지 않는 경우 스팸함을 확인하거나 이메일 주소를 다시 확인해주세요.
이미 가입한 적이 있다면 아이디 찾기 또는 비밀번호 찾기를 이용해주세요.
```

```text
이메일 인증이 완료되었습니다. 입력 오류가 있더라도 인증은 유지됩니다.
```

## 7. 다섯 번째 작업: 내 정보 수정 안내 카드 추가

### 문구 후보

```text
회원님의 닉네임과 이메일을 변경하실 수 있습니다.
닉네임 변경은 30일에 한 번만 가능합니다.
인증코드는 이메일 변경 시에만 입력해주세요.
```

## 8. 여섯 번째 작업: 마이페이지 언급 댓글 카드 가독성 개선

### 기대 수정 방향

- 선을 과도하게 추가하지 않는다.
- 컬럼 간 간격, 최소 폭, 정렬을 조정한다.
- 제목/본문 미리보기/날짜 영역이 시각적으로 구분되도록 한다.
- 모바일에서는 세로 카드형 전환 또는 자연스러운 줄바꿈을 유지한다.
