# 가힣 최종 배포 전 수동 회귀 테스트 체크리스트

## 사용 방법

- 테스트 환경: `prod-like MySQL validate` 환경 우선.
- 테스트 브라우저: Chrome 기준.
- 각 항목은 `[ ]`를 `[x]`로 바꾸며 진행한다.
- 실패 항목은 `04-test-result-template.md`의 이슈 기록표에 남긴다.
- 모든 기능을 완벽히 다 테스트하기 어렵다면 `P0`, `P1`을 먼저 끝낸다.

## 테스트 계정 준비

| 역할 | 권장 계정 | 목적 |
| --- | --- | --- |
| 관리자 | 운영 관리자 계정 | 관리자 화면/조치 테스트 |
| 일반 회원 A | 신규 가입 또는 테스트 회원 | 작성자 역할 |
| 일반 회원 B | 신규 가입 또는 테스트 회원 | 신고자/타 사용자 역할 |
| 정지 회원 | 관리자 조치로 생성 | 상태 제한 테스트 |
| 탈퇴 유예 회원 | 탈퇴 신청으로 생성 | 복구/접근 제한 테스트 |
